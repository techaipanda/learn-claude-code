### 问题：Python 外部管理环境错误

运行 `pip install -r requirements.txt` 时遇到：

```
error: externally-managed-environment
× This environment is externally managed
```

**原因**：macOS 使用 Homebrew 管理的 Python 不允许直接使用 pip 安装到系统环境。

### 解决方案

使用虚拟环境隔离项目依赖：

```bash
# 1. 创建虚拟环境
python3 -m venv .venv

# 2. 激活虚拟环境
source .venv/bin/activate

# 3. 安装依赖
pip3 install -r requirements.txt
```

### 重要：.env 配置

`.env` 文件必须只包含必要的配置项，不能有注释混杂在配置中：

```
ANTHROPIC_API_KEY=你的API key
MODEL_ID=MiniMax-M2.7
ANTHROPIC_BASE_URL=https://api.minimaxi.com/anthropic
```

### 常见问题

1. **ModuleNotFoundError: No module named 'anthropic'**
   - 原因：用的是系统 Python 而非虚拟环境
   - 解决：用 `python3` 或完整路径调用 `.venv/bin/python`

2. **API Key 无效**
   - 检查 `.env` 文件中 `ANTHROPIC_API_KEY` 是否正确
   - 确保没有多余的注释或空格

### VSCode 调试 Python

**方法 1：右键调试**
- 在 `s01_agent_loop.py` 文件上**右键**
- 选择 **"Debug Python File in Terminal"**

**方法 2：调试面板**
1. 点击 VSCode 左侧 **Run and Debug**（虫子图标）
2. 点击 **"Run and Debug"** 按钮
3. 选择 **"Python File"**

**方法 3：断点调试**
1. 在代码行号左侧点击设置断点（红点）
2. 按 `F5` 启动调试
3. 程序会在断点处暂停，可检查变量

**确保使用虚拟环境中的 Python**
1. 按 `Cmd + Shift + P`
2. 输入 `Python: Select Interpreter`
3. 选择 `.venv/bin/python`
